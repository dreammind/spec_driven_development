# 実装計画

1) 仕様確定
- コマンド一覧、引数/オプション、出力フォーマットの定義
- 使用言語は Python とする
- 開発手法はテスト駆動型開発（TDD）とする
- CLIのコマンド仕様表（例・入力・出力）を作成する

2) データ設計
- Taskスキーマ、優先度・日付形式・カテゴリ制約の決定

3) 基盤実装
- 解析: CLI引数パーサ
- 永続化: JSONの読み書き

4) コア機能
- add / list / done / undo / delete / edit / search
- ステップバイステップで、各機能ごとにテストを先に書いてから実装コードを書く
- 各機能のテストと実装が完了するたびに確認を行う
- 基本I/Oエラー（ファイルなし/壊れたJSON）のTDDを含める

5) 表示/操作性
- 絞り込み、並び替え、ページング（必要なら）

6) バリデーション強化
- 入力不備や不正値のエラーメッセージ整備

7) テスト実装
- ユースケース、エッジケース、I/O失敗時
- 境界値（空文字・長文・不正日付・カテゴリ空）のテストを含める

8) カバレッジ調整
- 90%達成まで追加テスト
- カバレッジ計測方法と合否基準を明記する

9) ドキュメント
- 使い方、例、想定エラー
