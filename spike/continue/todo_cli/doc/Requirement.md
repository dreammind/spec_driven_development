# タスク管理（TODO）CLIアプリ 要件定義書

## 1. 概要
CLI（コマンドラインインターフェース）上で動作するタスク管理（TODO）アプリケーションを開発する。本アプリは、ユーザーが日々のタスクを簡単に管理できることを目的とする。

## 2. 目的
- ユーザーがタスクを効率的に追加・管理・完了・削除できる環境を提供する。
- タスクの優先度、期限、カテゴリによる分類を可能にし、タスク管理の利便性を高める。

## 3. 機能要件
### 3.1 タスクの追加
- ユーザーは新しいタスクを追加できる。
- タスクには以下の情報を設定できる：
  - タスク名（必須）
  - 詳細（任意）
  - 優先度（例：高・中・低、必須）
  - 期限（任意）
  - カテゴリ（任意、複数可）

### 3.2 タスクの一覧表示
- 登録されているタスクを一覧表示できる。
- 一覧は以下の条件でソート・フィルタできる：
  - 優先度
  - 期限
  - カテゴリ
  - 完了・未完了状態

### 3.3 タスクの完了
- タスクを完了状態に変更できる。
- 完了済みタスクは一覧で区別して表示される。

### 3.4 タスクの編集
- 既存のタスクの内容を編集できる。
- 編集可能な項目は以下とする：
  - タスク名
  - 詳細
  - 優先度
  - 期限
  - カテゴリ
- 編集履歴の管理は必須要件としない。

### 3.5 タスクの削除
- タスクを削除できる。
- 完了・未完了に関わらず削除可能とする。

### 3.6 データ保存
- タスクデータはローカルストレージ（`tasks.json`）に保存される。
- アプリの再起動後もデータが保持されること。

## 4. 非機能要件
- 外部データベースは使用しない。
- 型チェックによる安全性を確保する。
- テストを導入し、テストカバレッジ90%以上を目標とする。

## 5. 想定ユーザー
- CLI操作に慣れた一般ユーザー
- タスク管理を効率化したい個人

## 6. 除外事項
- Web UIやモバイルアプリとしての提供は対象外とする。
- 外部サービスやAPIとの連携は行わない。

## 7. 用語定義
- タスク：ユーザーが管理したい作業や予定の単位。
- 優先度：タスクの重要度や緊急度を示す指標。
- 期限：タスクの完了を目指す日時。
- カテゴリ：タスクを分類するためのラベル。
